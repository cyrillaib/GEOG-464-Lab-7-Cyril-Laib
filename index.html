Written Answers

Q2. In styleAll(feature, latlng) we build a plain JavaScript object called styles that contains Leaflet symbol properties (e.g., stroke, color, fillColor, radius). The function then checks the GeoJSON geometry type with if (feature.geometry.type === "Point"). When the condition is true, it overrides some defaults (turns stroke on, sets a fill color/opacity, and a circle radius). Finally, the function returns the styles object; Leaflet reads those keys and applies them to the corresponding feature’s layer.

Q4. In onEachFeature: addPopups, the feature argument is the raw GeoJSON Feature (its geometry and properties). The layer argument is the Leaflet layer that was created for that feature (e.g., a L.CircleMarker for points). You use feature to read attributes and layer to style and bind events/pop-ups.

Q5. layer.options.fill is a data property (state/config on the Leaflet layer). layer.getLatLng() is a method (a function) that returns a computed LatLng object. One is static configuration, the other executes code to return a value.

Q6. Setting layer.options.fill = false removes the fill, so the circle becomes a hollow ring. Setting layer._radius = 80 (for L.CircleMarker) makes the marker very large. After reloading with those lines active, you’ll see unfilled, oversized markers.
